# 实时情报与知识库设计

MealMind 采用“固定知识库 + 今日动态情报区”的结构，参考世界杯 Skill 的设计。

## 固定知识库

- 循证饮食规则：WHO 健康饮食、地中海式饮食、饮食质量与心情研究。
- 用户安全规则：不做医疗诊断、不极端节食、不自动点餐付款。
- 推荐方法论：目标、营养、价格、距离、天气、心情、口味、数据可信度。

## 今日动态情报区

每天或每次运行时更新：

- 住址地理编码结果。
- 附近餐馆、便利店、超市、饮品店。
- 距离、步行时间、营业状态。
- 高德 POI 返回的人均、评分、招牌菜、照片、网址和潜在菜单字段。
- 超市价格、便利店商品、优惠是否由授权来源返回；缺失时明确说明。
- 天气、温度、降雨、空气质量。
- 用户当天心情、饥饿、压力、运动、预算倾向。

## 推荐接入顺序

1. V1：接入地图 API 获取 POI、距离、人均、评分、招牌菜。
2. V2：接入天气 API，把雨天、温度、空气质量写入动态情报区。
3. V3：支持菜单/小票/价签 OCR，由用户授权上传后更新本地价格库。
4. V4：如有官方授权接口，再接商家菜单和优惠。

## 可选 API

- 高德地图：地理编码、周边搜索、路线、天气。
- 腾讯位置服务：地址解析、地点搜索、距离计算、天气。
- 百度地图：地点检索、路线规划、天气。

## 禁止项

- 不抓私人微信群。
- 不未授权抓美团、饿了么、大众点评。
- 不自动点餐或付款。
- 不把搜索结果伪装成已验证事实。
