# 系统设计

MealMind 把日常吃饭拆成五层：固定用户画像、固定知识库、本地/实时情报、当天状态、推荐决策。

MVP 以授权地图 POI 为主要实时情报来源。没有实时 POI、用户授权文件或官方价格源时，系统不生成示例店铺推荐，避免把 demo 菜单伪装成真实结果。

核心流程：

1. 加载用户画像。
2. 加载循证饮食规则和安全边界。
3. 通过地图 Provider 查询附近餐馆、便利店、超市、饮品店。
4. 读取 POI 中的人均、评分、招牌菜、照片、网址和潜在菜单字段。
5. 明确标注菜单价、活动、超市菜价是否缺失。
6. 基于真实 POI 情报生成候选餐食。
7. 使用固定权重评分。
8. 输出结构化 JSON 和中文总结。

设计重点是“具体决策”：每次都要输出吃什么、喝什么、去哪买、多少钱、为什么。
